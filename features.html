What is "My Podcasts"?

Create and host complete podcasts directly on your server. Upload audio files, manage episodes, and generate professional RSS feeds for distribution to Apple Podcasts, Spotify, and other platforms.

How to Create a Podcast:

  1. Click "My Podcasts" button in the header
  2. Click "Create New Podcast"
  3. Fill in podcast details:
    • Basic info (title, description, author, email)
    • Cover image (1400-3000px square)
    • iTunes metadata (category, language, explicit flag)
  4. Click "Create Podcast"

How to Add Episodes:

  1. From your podcast card, click "Episodes"
  2. Click "Add New Episode"
  3. Upload MP3 audio file (up to 500MB)
  4. Add episode details (title, description, duration)
  5. Optional: Upload episode artwork
  6. Set publication date and status
  7. Click "Add Episode"

Your RSS Feed:

  • Each podcast gets its own RSS feed URL
  • iTunes-compliant (RSS 2.0 + iTunes namespace)
  • Submit to Apple Podcasts, Spotify, etc.
  • Optionally import into your main directory

Features:

  • Audio Hosting: Upload and host MP3 files on your server
  • Episode Management: Add, edit, delete episodes with full control
  • Cover Images: Podcast and episode artwork support
  • iTunes Compliance: Professional RSS feeds ready for distribution
  • Persistent Storage: All files stored safely in persistent volumes
üí° Pro Tip: After creating your podcast, use "Import from RSS" to add it to your main directory for easy browsing alongside your other podcasts!

üîÑ Clone from RSS - Download & Host Podcasts

What is Podcast Cloning?

Unlike RSS import (which links to external feeds), cloning downloads and hosts entire podcasts on your server. All audio files are stored locally, giving you full control.

How to Clone a Podcast:

  1. Go to "My Podcasts" page
  2. Click "Clone from RSS" button
  3. Enter the RSS feed URL
  4. Click "Validate Feed" to preview episode count and storage needs
  5. Click "Start Cloning" and wait (don't close window)
  6. Time estimates: 5-10 episodes (1-2 min), 20-50 episodes (3-5 min), 100+ episodes (10-30 min)

What Gets Cloned:

  • Podcast metadata (title, description, author, category)
  • Podcast cover image
  • All episode audio files (up to 500MB each)
  • Episode metadata (titles, descriptions, dates, durations)
  • Episode images (optional)
  • iTunes tags (episode numbers, seasons, explicit flags)

Technical Details:

  • Supported formats: MP3 and M4A (works with Anchor.fm/Spotify podcasts)
  • Max file size: 500MB per episode
  • Timeout: 10 minutes per file
  • Error recovery: Continues if some episodes fail
  • Storage: Files saved to uploads/audio/[podcast_id]/
  • RSS: Generates iTunes-compliant RSS 2.0 feed
‚ö†Ô∏è Important:
  • Ensure adequate disk space before cloning
  • Don't close browser window during cloning
  • Only clone content you have rights to host
  • Avoid multiple simultaneous clones (resource-intensive)
üí° Common Uses:
  • Archive podcasts before they're removed
  • Create backups of your own shows
  • Migrate content between hosting platforms
  • Build offline podcast libraries
  • Preserve educational content

üì¢ Banner Ads Manager - Monetize Your Platform

What is the Ads Manager?

Upload and manage banner advertisements for your web app and mobile feed. Display rotating banners on your homepage and serve ads to mobile apps via RSS feed.

How to Access:

  • Click "Ads Manager" button in the header
  • Or visit /ads-manager.php directly

Three Ad Types:

  • Web Banners (728x90px) - Display on homepage with rotation
  • Phone Banners (320x50px) - Small mobile banners for apps
  • Tablet Banners (728x90px) - Larger mobile banners for tablets

How to Upload Banners:

  1. Go to Ads Manager page
  2. Choose section (Web, Phone, or Tablet)
  3. Drag image to upload zone or click to browse
  4. System validates exact dimensions automatically
  5. Banner appears in grid and live preview

Add Destination URLs:

  1. Click "Add URL" button on any banner card
  2. Enter destination URL in modal (e.g., https://example.com/landing-page)
  3. Click "Save URL"
  4. Banner becomes clickable, opens in new tab
  5. Button changes to "Edit URL" for future updates

Manage Banners:

  • Delete - Click X button, confirm deletion
  • Section Toggle - Enable/disable entire web or mobile ad sections
  • Individual Toggle - Enable/disable each ad individually (toggle on date row)
  • Adjust Timing - Set rotation duration (5-60s) and fade duration (0.5-3s)

Per-Ad Enable/Disable:

  • Each banner has its own toggle switch (bottom right, on date row)
  • Toggle ON (green) = Ad appears in preview and feeds
  • Toggle OFF (gray) = Ad is hidden, shown at 50% opacity in manager
  • Changes are instant - preview updates in real-time
  • Only enabled ads appear in mobile RSS feed

Live Preview:

  • See exactly how banners will rotate on your site
  • Preview updates in real-time as you adjust settings
  • Click banners to test destination URLs
  • Single ad detection (no rotation for 0-1 ads)

Front-End Display:

  • Banner appears under header on public homepage (index.php)
  • Rotates automatically based on your settings
  • Clickable if URL is set
  • Only shows when enabled in Ads Manager
  • Responsive design (scales on mobile)

Mobile RSS Feed:

  • URL: https://your-domain.com/mobile-ads-feed.php
  • Copy URL from Ads Manager interface
  • Use in mobile/tablet apps for ad consumption
  • Includes dimensions, URLs, and display order
  • Standard RSS 2.0 format with custom ads namespace

  • Web: Exactly 728x90 pixels
  • Phone: Exactly 320x50 pixels
  • Tablet: Exactly 728x90 pixels

üìä Engagement Analytics - Track Your Audience

What is Engagement Analytics?

Monitor how your audience interacts with your podcasts. Track plays and downloads from the public player with beautiful visualizations and detailed metrics.

Key Features:

  • Overview Metrics: Total plays, downloads, unique listeners, and download conversion rate
  • Interactive Charts: Daily trend visualization with Chart.js (7d, 30d, 90d, all-time views)
  • Top Content: Top 10 episodes and podcasts ranked by engagement
  • Podcast Filtering: Use the dropdown to view analytics for individual podcasts
  • Privacy-Focused: No personal information collected, random session IDs only

How It Works:

  • Analytics are tracked only from the public player on index.php
  • Each episode is counted once per session (prevents duplicate tracking)
  • Sessions rotate every 24 hours for accurate unique listener counts
  • Play button, download button, and next/previous controls all trigger tracking
